## 核心安全边界

- 首次填写姓名和学号只代表“个人信息已锁定”，不代表学校身份核验；写错后必须提交修改申请。
- 只有姓名、学号唯一一致且卡片已经到达官方地点，失主确认后才会看到地点和短时环境照片地址。
- 多条匹配或个人保管状态不返回地点、照片地址或文件编号。
- 个人保管的卡片必须先转交官方地点，不提供陌生人直接联系或见面功能。
- 校园卡原图完成 OCR 后立即安排删除；环境照片和取卡照片按页面告知的期限进入可靠清理队列。
- 免费云环境中，环境照片和取卡证明由云函数代为保存；小程序只拿到一次性凭证，不拿云文件 ID。
- 归还完成使用 `handovers/{claimId}` 防止重复计数，卡片、认领和失卡登记在数据库事务中一起结束。
