## 核心安全边界

- 首次填写姓名和学号只代表“个人信息已锁定”，不代表学校身份核验；写错后必须提交修改申请。
- 姓名、学号一致且失主确认最新发布的对应卡片后，有清晰存放环境照片或卡片已在官方地点时才会显示领取信息和短时照片地址。
- 同一姓名、学号存在多条记录时只展示最新发布的一条；没有清晰存放环境照片时不返回领取地点或照片。
- 个人保管的卡片只有在发布时提供清晰的存放环境照片后，才会在姓名学号一致并确认对应卡片后显示领取信息；否则需转交官方地点。系统不提供陌生人直接联系或见面功能。
- 校园卡原图完成 OCR 后立即安排删除；环境照片和取卡照片按页面告知的期限进入可靠清理队列。
- 免费云环境中，环境照片和取卡证明由云函数代为保存；小程序只拿到一次性凭证，不拿云文件 ID。
- 归还完成使用 `handovers/{claimId}` 防止重复计数，卡片、认领和失卡登记在数据库事务中一起结束。
