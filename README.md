# 校园卡守护者

中国人民大学校园卡智能匹配与归还微信小程序。当前版本已配置正式 AppID；未填写云环境 ID 时自动使用本机演示数据，填写并部署云函数后自动切换到云端数据。

## 本地运行

1. 执行 `npm install`。
2. 打开微信开发者工具，导入本目录。
3. 执行 `npm run verify` 检查测试、类型、代码风格和格式。
4. 需要接入云端时，按 `docs/cloud-setup.md` 操作。

## 云端开通（需要项目管理员本人操作）

1. 在开发者工具中开通云开发并创建开发环境。
2. 将环境 ID 填入 `miniprogram/config/cloud.ts`。
3. 开通腾讯云 OCR，创建仅允许调用 OCR 的最小权限子账号。
4. 在云函数环境变量中配置 `STUDENT_HMAC_SECRET`、`TENCENT_SECRET_ID`、`TENCENT_SECRET_KEY`；禁止写入源码。
5. 按 `docs/cloud-setup.md` 创建集合、索引和权限。
6. 将受控管理员账号的 `users.role` 设置为 `admin`，由管理员处理重复匹配和现场交接。

## 核心流程

1. 用户首次保存姓名和学号后，系统锁定这两项信息并用于找卡。
2. 姓名、学号一致且只有一条校园卡记录时，系统直接确认；卡片在官方交卡点时立即显示地点。
3. 同一姓名和学号匹配到多条校园卡记录时，才进入管理员核对流程。
4. 个人暂时保管的位置不公开，由认领流程安排交接。
5. 现场确认归还后，卡片、认领申请和失卡登记在同一事务中同步结束。

## 隐私原则

- 原图只用于 OCR 与脱敏，成功后立即删除。
- 学号使用服务端 HMAC 匹配，不作为公开字段保存。
- 同一学号只能绑定一个微信账号，查询和认领必须与账号保存的姓名、学号同时一致。
- 公开列表只展示脱敏姓名、脱敏学号与脱敏图片。
- 官方交卡点在唯一匹配后展示；个人保管地点始终不在公开查询结果中展示。
- AI 审核能力默认关闭，只有管理员配置预算后才能启用。
