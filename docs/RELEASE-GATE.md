# 发布硬门禁

以下项目未全部人工确认时，不得把 0.2.0 及后续版本上传为体验版或提交审核。

- [ ] 在微信公众平台轮换截图中曾公开的测试号 AppSecret。新值仅保存在平台或部署环境变量中，不写入源码、Git、截图或聊天。
- [ ] 在云开发控制台应用 `security/storage.rules.json` 对应的云存储规则，确认敏感目录客户端不可读取、只有文件创建者可以写入；同时真机确认客户端和云函数都拒绝非本人目录。
- [ ] 在云开发控制台应用 `security/database.rules.json`，确认小程序客户端不能直接读写业务集合，所有业务操作只能经过云函数权限检查。
- [ ] 用未登录账号验证不能取得文件编号或下载图片。
- [ ] 用普通但未确认卡片的账号验证，即使猜到文件编号也不能下载。
- [ ] 用出现多条匹配的失主账号验证，不能取得地点、照片地址或文件编号。
- [ ] 用正确认领者验证，仅在卡片到达官方地点并确认后获得短时图片地址；地址到期后失效。
- [ ] 用管理员账号验证可以在争议处理中临时查看取卡证明，其他角色不能查看。
- [ ] 分别使用拾卡者、失主、管理员三个真机账号完成发布、匹配、官方转交、领取拍照、归还和风险复核。
- [ ] 确认正式云服务故障时页面进入只读错误态，没有产生任何本机演示记录。
- [ ] 确认校园卡原图已删除，环境照片和取卡证明进入约定期限的清理任务，失败任务能自动重试。
- [ ] GitHub Actions、TypeScript、ESLint、Prettier、单元测试和依赖风险门禁全部通过。

说明：仓库中的规则文件是控制台配置的受版本管理副本；是否真正生效必须以上述四类账号实测为准。
