# 发布硬门禁

以下项目未全部人工确认时，不得把 0.2.0 及后续版本上传为体验版或提交审核。

- [ ] 在微信公众平台轮换截图中曾公开的测试号 AppSecret。新值仅保存在平台或部署环境变量中，不写入源码、Git、截图或聊天。
- [ ] 免费环境的云存储权限保持“仅创建者可读写”，不得改成“所有用户可读”或“所有用户小可读写”。
- [ ] 真机确认存放环境照片和取卡证明由 `uploadPrivateImage` 保存：小程序只得到一次性上传凭证，接口响应和页面数据中均没有文件 ID。
- [ ] 真机确认普通上传者不是长期照片的云存储创建者；即使通过调试得到或猜到文件 ID，也不能直接下载。
- [ ] 在云开发控制台应用 `security/database.rules.json`，确认小程序客户端不能直接读写业务集合，所有业务操作只能经过云函数权限检查。
- [ ] 用未登录账号验证不能取得文件编号或下载图片。
- [ ] 用普通但未确认卡片的账号验证，即使猜到文件编号也不能下载。
- [ ] 用存在多条相同姓名学号记录的失主账号验证，只返回最新发布的一条记录及其最新照片。
- [ ] 用正确认领者验证，仅在姓名学号一致且有存放环境照片或卡片已在官方地点后获得短时图片地址；地址到期后失效。
- [ ] 用管理员账号验证可以在争议处理中临时查看取卡证明，其他角色不能查看。
- [ ] 分别使用拾卡者、失主、管理员三个真机账号完成发布、匹配、查看存放照片或官方地点、领取拍照、归还和风险复核；另行验证无照片记录不会泄露具体地点。
- [ ] 确认正式云服务故障时页面进入只读错误态，没有产生任何本机演示记录。
- [ ] 确认校园卡原图已删除，环境照片和取卡证明进入约定期限的清理任务，失败任务能自动重试。
- [ ] 确认 `scheduledCleanup` 会分页清理超过60天的 `auditLogs`。
- [ ] 确认数据删除审批会检查进行中认领、争议和照片保留期，并生成不含姓名、学号、OpenID 的 `deletionReceipts`。
- [ ] 确认本版本没有 `getPhoneNumber`、手机号收集文案或手机号发布/认领门槛。
- [ ] 核对首页、注意事项、隐私说明均标明个人公益项目、非官方、运营者、邮箱、生效日期和规则版本。
- [ ] 核对 found/lost/claim/thanks/general 举报入口、限流、管理员三种决定及站内通知。
- [ ] GitHub Actions、TypeScript、ESLint、Prettier、单元测试和依赖风险门禁全部通过。

说明：免费环境不能启用仓库中的自定义云存储规则，因此采用“仅创建者可读写 + 云函数代存长期照片”。`security/storage.rules.json` 是将来升级套餐后的规则副本；免费方案是否真正隔离仍必须以上述四类账号实测为准。
